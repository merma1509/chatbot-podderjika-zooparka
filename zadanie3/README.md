# Task 3 - LoRA Training Dataset Generation

## 1. Источники данных

### 1.1 Основные источники

- **Базовые промпты из Задания 2**: 10 adversarial промптов покрывающих 9 типов атак
- **Синтетическая генерация**: Алгоритмическое расширение до 1000+ обучающих примеров
- **Вариации на основе шаблонов**: Систематическая промпт-инженерия для разнообразия

### 1.2 Покрытие типов атак

- **Prompt Injection**: Прямые попытки манипуляции системой
- **Input Corruption**: Атаки на форматирование и кодирование текста
- **Data Integrity Attack**: Внедрение ложной информации
- **Policy Extraction**: Пробирование внутренних правил и руководств
- **Hallucination Induction**: Создание ложной реальности
- **Multi-intent Confusion**: Одновременные множественные запросы
- **Retrieval Boundary Attack**: Тестирование пределов контекста
- **Role-play Attack**: Эксплуатация на основе персонажей
- **Social Engineering**: Психологическая манипуляция

## 2. Предобработка и очистка данных

### 2.1 Нормализация ввода

- **Стандартизация текста**: Нормализация Unicode, очистка пробелов
- **Согласованность кодирования**: UTF-8 кодирование для всех датасетов
- **Валидация качества**: Удаление бессмысленных или поврежденных промптов

### 2.2 Валидация данных

- **Проверка атакующего вектора**: Убедиться что каждый промпт сохраняет adversarial намерение
- **Валидация языка**: Проверка согласованности русского языка и грамматики
- **Классификация безопасности**: Категоризация по уровню риска

### 2.3 Контекстуальное улучшение

- **Контекст сценария**: Добавление реалистичных контекстов разговора

## 3. Метод составления обучающей выборки

### 3.1 Стратегия сбалансированной выборки

```text
Всего примеров: 1000
Типы атак: 9 категорий × ~111 примеров каждая
Типы сценариев: 8 категорий × ~125 примеров каждая
Уровни сложности: Easy (30%), Medium (40%), Hard (30%)
```

### 3.2 Пайплайн генерации сценариев

1. **Выбор базового промпта**: Случайный выбор из промптов Задания 2
2. **Присвоение типа сценария**: Сбалансированное распределение по 8 типам сценариев
3. **Генерация вариаций**: Генерация промптов на основе шаблонов
4. **Создание шаблона ответа**: Создание ожидаемых шаблонов безопасных ответов
5. **Валидация качества**: Автоматическая проверка релевантности и безопасности

### 3.3 Типы сценариев

- **Direct Attack**: Прямые adversarial запросы
- **Subtle Manipulation**: Косвенные попытки влияния
- **Context Confusion**: Эксплуатация контекста разговора
- **Role Exploitation**: Попытки обхода на основе персонажей
- **Boundary Testing**: Пробирование пределов безопасности
- **Multi-intent**: Сложные одновременные запросы
- **Information Extraction**: Попытки доступа к чувствительным данным
- **Behavior Manipulation**: Изменение шаблонов ответа

## 4. Обеспечение баланса обучающей выборки

### 4.1 Балансировка по типам атак

### 4.2 Балансировка по уровням сложности

- **Easy (30%)**: Простые, прямые атаки с ясным вредоносным намерением
- **Medium (40%)**: Умеренно сложные атаки требующие некоторого анализа

### 4.3 Разнообразие сценариев

- **Вариации шаблонов**: Множество паттернов формулировки для каждого сценария

### 4.4 Покрытие поведенческих сценариев

- **Тренировка сопротивления**: Промпты которые должны быть отклонены
- **Тренировка обнаружения**: Промпты требующие обнаружения манипуляции
- **Тренировка уточнения**: Промпты требующие уточнения контекста
- **Принуждение границ**: Промпты тестирующие системные пределы
